# Plan 5.1: Containerization

## Objective
Dockerize the Panopticon frontend application.
Purpose: To ensure consistent environments across development and production and to meet the final Roadmap requirement.
Output: A multi-stage `Dockerfile`, an optimized `.dockerignore`, and `compose.yml` (production) and `compose.dev.yml` (development) files.

## Context
- .gsd/SPEC.md
- .gsd/ROADMAP.md
- .gsd/DECISIONS.md

## Tasks

<task type="auto">
  <name>Create .dockerignore</name>
  <files>.dockerignore</files>
  <action>
    Create a `.dockerignore` file in the project root.
    Exclude `node_modules`, `.git`, `.gsd`, `build`, `dist`, `.env`, and other unnecessary files to keep the Docker context small.
  </action>
  <verify>Check `.dockerignore` file existence and content</verify>
  <done>`.dockerignore` correctly excludes unnecessary directories</done>
</task>

<task type="auto">
  <name>Create multi-stage Dockerfile</name>
  <files>Dockerfile</files>
  <action>
    Create a multi-stage `Dockerfile`.
    - Stage 1 (builder): Use `node:20-alpine` (or 22), copy `package.json` and `package-lock.json`, run `npm ci`, copy the rest of the files, and run `npm run build`.
    - Stage 2 (production): Use a lightweight base image, copy the built assets from the builder stage, and expose port 3000. Implement the start command (`npm start` or equivalent depending on the React Router setup).
    AVOID: Pushing development dependencies into the final image.
  </action>
  <verify>Run `docker build -t panopticon-frontend .`</verify>
  <done>Image builds successfully without errors</done>
</task>

<task type="auto">
  <name>Create compose.yml and compose.dev.yml</name>
  <files>compose.yml, compose.dev.yml</files>
  <action>
    Create `compose.dev.yml` for local development. Mount the source code as a volume for hot-reloading, expose port `3000`, and start the app with `npm run dev`.
    Create `compose.yml` for production. Define a `frontend` service that builds from the current directory `.` and maps the exposed port to `3000:3000`.
    In both files, include a commented-out placeholder configuration for the backend service named `panoptic-api`.
  </action>
  <verify>Run `docker compose config` and `docker compose -f compose.dev.yml config` to validate structure</verify>
  <done>Docker Compose configurations are valid for both dev and prod</done>
</task>

## Success Criteria
- [ ] A multi-stage `Dockerfile` is present and functional.
- [ ] `compose.yml` successfully orchestrates the production container.
- [ ] `compose.dev.yml` successfully orchestrates the development container with volume mounts.
- [ ] The app can be started using Docker tools on port 3000.
